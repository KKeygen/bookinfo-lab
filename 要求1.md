实验目标
能够使用 docker 和 k8s 部署 bookinfo 微服务（yaml 部署和源码部署）。
背景知识
Docker
Docker 是一个开源的容器化平台，它允许开发者将应用程序及其所有依赖（代码、运行时、库、配置文件）打包成一个标准化的单元——容器（Container）。容器之间相互隔离，但共享宿主机的操作系统内核，因此比传统虚拟机更加轻量、启动更快。
在微服务架构中，Docker 扮演着关键角色：每个微服务被独立打包成一个镜像（Image），在各自的容器中运行，互不干扰。这使得服务的开发、测试、部署和扩缩容都可以独立进行，完全符合微服务"高内聚、低耦合"的设计原则。
核心概念：Image（镜像） 是只读模板；Container（容器） 是镜像的运行实例；Dockerfile 定义如何构建镜像；Registry（镜像仓库） 用于存储和分发镜像（如 Docker Hub）。

Kubernetes（k8s）
https://kubernetes.io/zh-cn/docs/home/
当微服务数量增多，手动管理大量容器变得不可行——你需要自动化地处理服务发现、负载均衡、故障恢复、滚动更新、弹性扩缩容等问题。Kubernetes（简称 k8s）正是为此而生的容器编排平台，由 Google 开源，目前是业界标准。
K8s 将若干台机器组成一个集群（Cluster），并以声明式的方式管理容器工作负载——你只需描述"期望状态"，k8s 会持续确保系统达到并维持该状态。
在本实验中，我们使用 Docker Desktop 内置的 k8s 支持（基于 kind），搭建一个 1 个控制节点 + 4 个工作节点的本地集群，将 bookinfo 的各个微服务分散部署在不同节点上。
核心概念：
- Node：集群中的一台机器（物理机或虚拟机）
- Pod：k8s 最小调度单元，通常封装一个容器
- Deployment：声明 Pod 的期望副本数和更新策略
- Service：为一组 Pod 提供稳定的网络访问入口
- Namespace：用于隔离集群内不同项目或环境的资源

四、通过源码部署 bookinfo 服务
源码部署的本质其实是将源码打包成 docker 镜像，将 docker 镜像 push 到 docker hub，部署到 k8s 上再从 docker hub 上面将容器拉取下来完成部署。
4.1 申请阿里云容器镜像服务
为了避免网络问题，我们使用国内的 docker hub（例如阿里云的容器镜像服务）。
访问：https://cr.console.aliyun.com/cn-hangzhou/instances
选择创建个人实例，创建成功后首页如下图所示，注意仓库信息里的公网 url，同时记住之前创建时输入的密码，后面会用到。
[图片]
依次点击“命名空间”->“创建命名空间”。
[图片]
这样就完成了阿里云 docker hub 的开通。
另外，请将自动创建仓库默认配置设置为“公开”，这样后续会省去拉取镜像时的身份验证的麻烦（这只是方便我们学习，生产环境千万不能这么做！）。
回到本地完成登录：
> docker login --username=[你的阿里云账号] [阿里云仓库公网 URL]
这样就能在本地访问我们的阿里云镜像服务了。
4.2 打包并上传镜像
首先克隆 book info 的官方仓库（其实是 istio 仓库里的 example）
> git clone https://github.com/istio/istio.git
进入到 book info 文件夹：
> cd istio/samples/bookinfo/src
可以发现这里有四个文件夹 detials、productpage、ratings、reviews，刚好对应 book info 的四个微服务。
[图片]
每个文件夹下都有一个 DockerFile 文件，因此分别进入对应的文件夹，将源代码打包成镜像。这里以 details 为例，实际执行的时候，四个微服务都要进行对应的打包和上传。
> cd details
> docker build -t [镜像名称]:[镜像版本] .  # 不要忘记最后的 . 它表示当前目录
然后我们将其关联到阿里云的镜像仓库
> docker tag [本地镜像名称]:[镜像版本] [阿里云仓库公网 URL]/[命名空间]/[镜像名称]:[镜像版本]
然后将其推送到阿里云的镜像仓库：
> docker push [阿里云仓库公网 URL]/[命名空间]/[镜像名称]:[镜像版本]
我们就可以在镜像仓库这个页面看到我们刚刚上传的镜像啦：
[图片]
4.3 将上传的镜像部署到本地的 k8s 集群
为了能让本地部署的微服务能拉取我们上传的镜像，需要修改 istio/samples/bookinfo/platform/kube/bookinfo.yaml 这个文件里面的内容。
具体怎么看这个 yaml 文件，请参考Kubernetes YAML 配置详解。
找到每个服务的 images 字段，将其改为我们的仓库地址：
apiVersion: apps/v1
kind: Deployment
metadata:
  name: details-v1
  labels:
    app: details
    version: v1
spec:
  replicas: 1
  selector:
    matchLabels:
      app: details
      version: v1
  template:
    metadata:
      labels:
        app: details
        version: v1
    spec:
      serviceAccountName: bookinfo-details
      containers:
      - name: details
        image: [自己上传之后的镜像地址]
        imagePullPolicy: Always
        ports:
        - containerPort: 9080
最后在 istio/samples/bookinfo/platform/kube文件夹下 apply 这个文件即可，后面的内容就和实验步骤三一样了。
kubectl apply -f .\bookinfo.yaml -n bookinfo-src
